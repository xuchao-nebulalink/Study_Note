# 新增或修改其他项目：二创项目通用模板

> 以后你不管部署什么项目，开源项目、二创项目、自写项目，都按这个套路来。核心就是：单独目录、独立端口、独立 compose、Caddy 单独反代。

## 1. 新增一个项目的标准流程

假设项目叫：

```text
myapp
```

域名：

```text
myapp.xcwindfall.top
```

宿主机端口：

```text
18081
```

创建目录：

```bash
mkdir -p /opt/apps/myapp
cd /opt/apps/myapp
mkdir -p data logs backups
```

## 2. 如果项目有现成 Docker 镜像

创建 `.env`：

```bash
nano .env
```

写入：

```env
APP_PORT=18081
TZ=Asia/Shanghai
```

创建 `docker-compose.yml`：

```bash
nano docker-compose.yml
```

示例：

```yaml
services:
  myapp:
    image: 镜像名:latest
    container_name: myapp
    restart: unless-stopped
    ports:
      - "127.0.0.1:${APP_PORT:-18081}:容器内端口"
    environment:
      TZ: ${TZ:-Asia/Shanghai}
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
```

比如容器内是 3000：

```yaml
ports:
  - "127.0.0.1:${APP_PORT:-18081}:3000"
```

启动：

```bash
docker compose up -d
```

本机测试：

```bash
curl -I http://127.0.0.1:18081
```

Caddy：

```caddyfile
myapp.xcwindfall.top {
    reverse_proxy 127.0.0.1:18081
}
```

重载：

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

## 3. 如果是 GitHub 项目，需要本地构建

```bash
cd /opt/apps
git clone https://github.com/作者/项目.git myapp
cd myapp
```

如果项目自带 Dockerfile：

```yaml
services:
  myapp:
    build:
      context: .
      dockerfile: Dockerfile
    image: local/myapp:dev
    container_name: myapp
    restart: unless-stopped
    ports:
      - "127.0.0.1:18081:3000"
    env_file:
      - .env
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
```

构建启动：

```bash
docker compose up -d --build
```

## 4. 如果项目是 Node/React/Vue/Next.js

### 4.1 Dockerfile 示例

```dockerfile
FROM node:20-alpine AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci

FROM node:20-alpine AS runner
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

ENV NODE_ENV=production
EXPOSE 3000
CMD ["npm", "run", "start"]
```

compose：

```yaml
services:
  web:
    build: .
    container_name: myapp-web
    restart: unless-stopped
    ports:
      - "127.0.0.1:18081:3000"
    env_file:
      - .env
```

## 5. 如果项目是 Python/FastAPI

Dockerfile：

```dockerfile
FROM python:3.11-slim
WORKDIR /app

RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

```yaml
      - "127.0.0.1:18081:8000"
```

## 6. 如果项目需要 PostgreSQL

compose 示例：

```yaml
services:
  app:
    image: 镜像名:latest
    container_name: myapp
    restart: unless-stopped
    ports:
      - "127.0.0.1:18081:3000"
    environment:
      DATABASE_URL: postgresql://myapp:你的密码@postgres:5432/myapp
    depends_on:
      postgres:
        condition: service_healthy
    networks:
      - myapp-network

  postgres:
    image: postgres:15-alpine
    container_name: myapp-postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: myapp
      POSTGRES_PASSWORD: 你的密码
      POSTGRES_DB: myapp
    volumes:
      - ./postgres_data:/var/lib/postgresql/data
    networks:
      - myapp-network
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U myapp -d myapp"]
      interval: 10s
      timeout: 5s
      retries: 5

networks:
  myapp-network:
    driver: bridge
```

不要暴露：

```yaml
ports:
  - "5432:5432"
```

除非你明确要外部连数据库。

## 7. 如果项目需要 Redis

```yaml
services:
  app:
    image: 镜像名:latest
    environment:
      REDIS_URL: redis://:你的Redis密码@redis:6379
    depends_on:
      redis:
        condition: service_healthy
    networks:
      - myapp-network

  redis:
    image: redis:7-alpine
    container_name: myapp-redis
    restart: unless-stopped
    command: redis-server --appendonly yes --requirepass 你的Redis密码
    volumes:
      - ./redis_data:/data
    networks:
      - myapp-network
    healthcheck:
      test: ["CMD-SHELL", "redis-cli -a '你的Redis密码' ping | grep PONG"]
      interval: 10s
      timeout: 5s
      retries: 5
```

## 8. 修改已有项目端口

### 8.1 改 compose

原来：

```yaml
ports:
  - "127.0.0.1:8080:8080"
```

改成：

```yaml
ports:
  - "127.0.0.1:18080:8080"
```

### 8.2 改 Caddy

原来：

```caddyfile
app.xcwindfall.top {
    reverse_proxy 127.0.0.1:8080
}
```

改成：

```caddyfile
app.xcwindfall.top {
    reverse_proxy 127.0.0.1:18080
}
```

### 8.3 重启

```bash
cd /opt/apps/app
docker compose up -d
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

## 9. 二创项目怎么更新

比如你 fork 了别人的项目：

```bash
cd /opt/apps/myapp
git status
```

如果你改过代码，先提交：

```bash
git add .
git commit -m "my custom changes"
```

拉上游更新前，先添加上游仓库：

```bash
git remote add upstream https://github.com/原作者/项目.git
```

拉取：

```bash
git fetch upstream
```

合并：

```bash
git merge upstream/main
```

如果分支是 master：

```bash
git merge upstream/master
```

有冲突就解决冲突，然后重新构建：

```bash
docker compose up -d --build
```

## 10. 二创项目最稳更新法

如果你不熟 Git 冲突，建议：

```text
1. 先备份整个目录
2. 新建一个 myapp_new 目录拉最新版
3. 把 .env、data、配置文件复制过去
4. 用新端口启动测试
5. 测试没问题，再切 Caddy 到新端口
6. 旧目录保留几天，没问题再删
```

示例：

```bash
cd /opt/apps
tar czf myapp_backup_$(date +%Y%m%d_%H%M%S).tar.gz myapp

git clone https://github.com/作者/项目.git myapp_new
cd myapp_new
cp ../myapp/.env ./.env
cp -r ../myapp/data ./data
```

新项目先用临时端口：

```yaml
ports:
  - "127.0.0.1:19081:3000"
```

启动测试：

```bash
docker compose up -d --build
curl -I http://127.0.0.1:19081
```

确认没问题，再把 Caddy 从旧端口切到新端口。

## 11. 项目命名规则

建议统一：

```text
项目目录：小写中划线，例如 new-api-test、myapp-admin
容器名：项目名-服务名，例如 myapp-web、myapp-postgres
网络名：项目名-network
数据库名：项目名
数据目录：data、logs、postgres_data、redis_data
```

不要所有项目都叫：

```text
app
web
redis
postgres
```

容易冲突。
