# 1. 一个服务器多个项目

## 核心方案

一台服务器跑多个项目，推荐这样搞：

```text
公网访问
  ↓
域名 / 子域名
  ↓
Caddy：统一负责 HTTPS + 反向代理
  ↓
不同项目的本地端口
  ↓
每个项目各自 docker compose
```

原则：

```text
只开放 80/443 给外网
项目端口只绑定 127.0.0.1
数据库 Redis 不暴露外网
每个项目一个目录、一个 compose、一个备份目录
```

## 目录规划

```bash
sudo mkdir -p /opt/projects
sudo chown -R $USER:$USER /opt/projects

cd /opt/projects
mkdir -p newapi sub2api cliproxyapi backups
```

推荐结构：

```text
/opt/projects/
├── newapi/
│   ├── docker-compose.yml
│   ├── .env
│   ├── data/
│   ├── logs/
│   ├── postgres/data/
│   ├── redis/data/
│   └── backups/
├── sub2api/
│   ├── docker-compose.local.yml
│   ├── .env
│   ├── data/
│   ├── postgres_data/
│   ├── redis_data/
│   └── backups/
├── cliproxyapi/
│   ├── docker-compose.yml
│   ├── config.yaml
│   ├── auths/
│   ├── logs/
│   └── backups/
└── backups/
```

## 端口规划

| 项目 | 域名 | 宿主机端口 | 容器端口 |
|---|---|---:|---:|
| New API | `newapi.xcwindfall.top` | `3000` | `3000` |
| Sub2API | `sub2api.xcwindfall.top` | `8081` | `8080` |
| CLIProxyAPI | `cliproxy.xcwindfall.top` | `8317` | `8317` |
| 其他项目 | `xxx.xcwindfall.top` | `9001+` | 看项目 |

端口写法要这样：

```yaml
ports:
  - "127.0.0.1:3000:3000"
```

不要这样：

```yaml
ports:
  - "3000:3000"
```

原因：第二种会让项目端口直接暴露到公网，容易被扫。

## 安装基础环境

```bash
sudo apt update
sudo apt install -y ca-certificates curl gnupg lsb-release ufw nano unzip git
```

安装 Docker：

```bash
curl -fsSL https://get.docker.com | sudo bash
sudo usermod -aG docker $USER
newgrp docker
docker version
docker compose version
```

安装 Caddy：

```bash
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' \
  | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg

curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' \
  | sudo tee /etc/apt/sources.list.d/caddy-stable.list

sudo apt update
sudo apt install -y caddy
```

防火墙：

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status
```

## 启动项目的通用流程

进入项目目录：

```bash
cd /opt/projects/项目名
```

启动：

```bash
docker compose up -d
```

看状态：

```bash
docker compose ps
```

看日志：

```bash
docker compose logs -f
```

重启：

```bash
docker compose restart
```

停止：

```bash
docker compose down
```

## 常用检查命令

看端口占用：

```bash
sudo ss -lntp
```

只看指定端口：

```bash
sudo ss -lntp | grep 3000
```

本机测试项目是否通：

```bash
curl -I http://127.0.0.1:3000
curl -I http://127.0.0.1:8081
curl -I http://127.0.0.1:8317
```

看 Caddy 状态：

```bash
sudo systemctl status caddy
sudo journalctl -u caddy -f
```

## 最佳习惯

1. 不要所有项目都写到一个 `docker-compose.yml`。
2. 不要把 PostgreSQL、Redis 暴露到公网。
3. 不要多个项目抢同一个宿主机端口。
4. 每个项目更新前先备份。
5. Caddy 只负责域名 HTTPS 和反代，不要把业务服务直接放 Caddy 里。
