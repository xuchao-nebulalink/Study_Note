# 一个服务器多个项目部署笔记

> 核心方案：**每个项目一个独立目录 + 一个独立 docker-compose.yml + 宿主机只开放 80/443 + Caddy 统一反代**。

## 1. 推荐架构

```text
公网用户
  ↓
域名 / 子域名
  ↓
Caddy：统一监听 80/443，自动 HTTPS
  ↓
127.0.0.1:项目端口
  ↓
Docker Compose 项目容器
```

服务器上不要每个项目都抢 80/443。  
**80/443 只给 Caddy 用，其他项目都跑在 127.0.0.1:不同端口。**

## 2. 推荐目录结构

```bash
/opt/projects/
├── new-api/
│   ├── docker-compose.yml
│   ├── data/
│   └── logs/
├── sub2api/
│   ├── docker-compose.local.yml
│   ├── .env
│   ├── data/
│   ├── postgres_data/
│   └── redis_data/
├── cliproxyapi/
│   ├── docker-compose.yml
│   ├── config.yaml
│   ├── auths/
│   └── logs/
└── other-project/
    ├── docker-compose.yml
    └── data/
```

创建目录：

```bash
sudo mkdir -p /opt/projects
sudo chown -R $USER:$USER /opt/projects
```

## 3. 端口规划

| 项目 | 宿主机端口 | 访问域名 |
|---|---:|---|
| New API | 3000 | `https://newapi.xcwindfall.top` |
| Sub2API | 13000 | `https://sub2api.xcwindfall.top` |
| CLIProxyAPI 主接口 | 8317 | `https://cliproxyapi.xcwindfall.top` |
| 其他项目 A | 8081 | `https://a.xcwindfall.top` |
| 其他项目 B | 8082 | `https://b.xcwindfall.top` |

重点：端口不要重复。

查看端口占用：

```bash
sudo ss -tulpn | grep -E ':80|:443|:3000|:13000|:8317|:8081|:8082'
```

## 4. 每个项目的 compose 原则

推荐写法：

```yaml
ports:
  - "127.0.0.1:8081:8080"
```

意思是：

```text
只允许本机访问 8081，外网不能直接访问。
外网必须通过 Caddy 的 HTTPS 域名访问。
```

不推荐：

```yaml
ports:
  - "8081:8080"
```

这种写法会把端口直接暴露到公网，不安全。

## 5. Caddy 统一反代配置

编辑：

```bash
sudo nano /etc/caddy/Caddyfile
```

示例：

```caddyfile
newapi.xcwindfall.top {
    reverse_proxy 127.0.0.1:3000
}

sub2api.xcwindfall.top {
    reverse_proxy 127.0.0.1:13000
}

cliproxyapi.xcwindfall.top {
    reverse_proxy 127.0.0.1:8317
}

a.xcwindfall.top {
    reverse_proxy 127.0.0.1:8081
}

b.xcwindfall.top {
    reverse_proxy 127.0.0.1:8082
}
```

检查 Caddy 配置：

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
```

重载 Caddy：

```bash
sudo systemctl reload caddy
```

看 Caddy 状态：

```bash
sudo systemctl status caddy --no-pager
```

看日志：

```bash
journalctl -u caddy -f
```

## 6. 防火墙建议

只开放 SSH、HTTP、HTTPS：

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status
```

项目端口比如 3000、13000、8317 不需要开放到公网。

## 7. 常用命令

进入某个项目：

```bash
cd /opt/projects/new-api
```

启动：

```bash
docker compose up -d
```

停止：

```bash
docker compose down
```

更新镜像：

```bash
docker compose pull
docker compose up -d
```

看日志：

```bash
docker compose logs -f
```

看容器：

```bash
docker ps
```

## 8. 最佳实践

1. 一个项目一个目录。
2. 一个项目一个 `docker-compose.yml`。
3. 不同项目使用不同宿主机端口。
4. 宿主机只开放 80/443。
5. Caddy 统一负责 HTTPS 和反代。
6. 数据目录必须挂载出来，方便备份。
7. 项目端口尽量绑定 `127.0.0.1`，不要直接暴露公网。

## 9. 备份方式

备份单个项目：

```bash
cd /opt/projects
sudo tar -czvf new-api-backup-$(date +%F).tar.gz new-api
```

恢复：

```bash
cd /opt/projects
sudo tar -xzvf new-api-backup-2026-05-31.tar.gz
```
