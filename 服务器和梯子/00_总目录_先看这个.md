# 服务器多项目部署笔记总目录

> 适合你的场景：一台 Ubuntu 22.04 服务器，只有一个主域名 `xcwindfall.top`，以后要放多个项目：New API、Sub2API、CLIProxyAPI，以及各种二创/自写项目。

## 最推荐的总方案

```text
用户浏览器
   ↓
DNS：不同子域名都解析到同一台服务器 IP
   ↓
服务器 80/443：Caddy 统一接收 HTTPS 请求
   ↓
Caddy 根据域名转发到不同本地端口
   ↓
每个项目自己一个目录、自己一个 docker-compose.yml、自己一套数据目录
```

例如：

```text
newapi.xcwindfall.top       → 127.0.0.1:3000 → New API
sub2api.xcwindfall.top      → 127.0.0.1:8080 → Sub2API
cliproxy.xcwindfall.top     → 127.0.0.1:8317 → CLIProxyAPI
app1.xcwindfall.top         → 127.0.0.1:18081 → 你的其他项目
app2.xcwindfall.top         → 127.0.0.1:18082 → 你的二创项目
```

## 你的服务器建议目录

以后所有项目都放这里：

```bash
sudo mkdir -p /opt/apps
sudo chown -R $USER:$USER /opt/apps
```

最终目录大概这样：

```text
/opt/apps/
├── newapi/
│   ├── docker-compose.yml
│   ├── .env
│   ├── data/
│   ├── logs/
│   ├── postgres_data/
│   └── redis_data/
├── sub2api/
│   ├── docker-compose.local.yml
│   ├── .env
│   ├── data/
│   ├── postgres_data/
```

## 核心原则

### 1. 每个项目端口必须不同

比如：

```text
New API       → 127.0.0.1:3000
Sub2API       → 127.0.0.1:8080
CLIProxyAPI   → 127.0.0.1:8317
其他项目1      → 127.0.0.1:18081
其他项目2      → 127.0.0.1:18082
```

### 2. 数据库和 Redis 不要暴露公网

compose 里不要写这种：

```yaml
ports:
  - "5432:5432"
  - "6379:6379"
```

除非临时调试，并且最好只绑定本机：

```yaml
ports:
  - "127.0.0.1:5432:5432"
```

### 3. 对外只开放 80 和 443

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp
sudo ufw enable
```

项目端口例如 3000、8080、8317，尽量只绑定 `127.0.0.1`，不直接暴露公网。

### 4. 修改 Caddyfile 后必须检查再重载

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

### 5. 更新项目之前先备份

特别是 New API、Sub2API 这种带数据库的项目：

```bash
cd /opt/apps/项目目录
mkdir -p backups
# 先看对应项目笔记里的备份命令
```
