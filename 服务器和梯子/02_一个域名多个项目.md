# 一个域名多个项目部署笔记

> 你只有一个主域名也没问题。推荐用：**多个子域名对应多个项目**。

你的域名假设是：

```text
xcwindfall.top
```

推荐这样分：

```text
newapi.xcwindfall.top       → New API
sub2api.xcwindfall.top      → Sub2API
cliproxyapi.xcwindfall.top  → CLIProxyAPI
blog.xcwindfall.top         → 博客
app.xcwindfall.top          → 其他项目
```

## 1. 最推荐：子域名方式

优点：

1. 最清晰。
2. 每个项目互不影响。
3. Caddy 配置简单。
4. 后续迁移方便。
5. 很多项目对路径部署支持不好，用子域名最稳。

示例：

```text
https://newapi.xcwindfall.top
https://sub2api.xcwindfall.top
https://cliproxyapi.xcwindfall.top
```

## 2. DNS 怎么写

去你的域名解析后台，添加 A 记录。

### 方式 A：逐个添加

| 主机记录 | 类型 | 值 |
|---|---|---|
| `newapi` | A | 你的服务器公网 IP |
| `sub2api` | A | 你的服务器公网 IP |
| `cliproxyapi` | A | 你的服务器公网 IP |
| `app` | A | 你的服务器公网 IP |

### 方式 B：泛解析

| 主机记录 | 类型 | 值 |
|---|---|---|
| `*` | A | 你的服务器公网 IP |
| `@` | A | 你的服务器公网 IP |

泛解析的意思是：

```text
任何 xxx.xcwindfall.top 都指向这台服务器
```

更适合长期一个服务器部署很多项目。

## 3. Caddy 怎么写

```caddyfile
newapi.xcwindfall.top {
    reverse_proxy 127.0.0.1:3000
}

sub2api.xcwindfall.top {
    reverse_proxy 127.0.0.1:13000
}

cliproxyapi.xcwindfall.top {
    reverse_proxy 127.0.0.1:8317
}

app.xcwindfall.top {
    reverse_proxy 127.0.0.1:8081
}
```

改完检查并重载：

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

## 4. 不推荐：路径方式

比如：

```text
https://xcwindfall.top/newapi
https://xcwindfall.top/sub2api
https://xcwindfall.top/cliproxyapi
```

不推荐原因：

1. 很多前端项目默认根路径是 `/`，放到 `/newapi` 容易白屏。
2. API 路径容易冲突。
3. WebSocket、回调地址、静态资源路径容易出问题。
4. 每个项目都要额外适配 base path。

所以除非项目明确支持 `BASE_URL=/xxx`，否则别用路径方式。

## 5. 一个主域名 + 多个子域名的最终形态

```text
xcwindfall.top
├── newapi.xcwindfall.top       → 127.0.0.1:3000
├── sub2api.xcwindfall.top      → 127.0.0.1:13000
├── cliproxyapi.xcwindfall.top  → 127.0.0.1:8317
├── app.xcwindfall.top          → 127.0.0.1:8081
└── admin.xcwindfall.top        → 127.0.0.1:8082
```

## 6. 排查命令

检查域名是否解析到服务器：

```bash
ping newapi.xcwindfall.top
```

或者：

```bash
nslookup newapi.xcwindfall.top
```

服务器本机测试项目端口：

```bash
curl -I http://127.0.0.1:3000
curl -I http://127.0.0.1:13000
curl -I http://127.0.0.1:8317
```

测试 HTTPS：

```bash
curl -I https://newapi.xcwindfall.top
curl -I https://sub2api.xcwindfall.top
curl -I https://cliproxyapi.xcwindfall.top
```

## 7. 结论

长期最稳方案：

```text
一个服务器
一个主域名
多个子域名
一个 Caddy
多个 Docker Compose 项目
```

不要每个项目都装一个 Caddy，也不要每个项目都抢 80/443。
