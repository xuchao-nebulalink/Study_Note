# 2. 一个域名多个项目

## 推荐做法

你只有一个主域名，比如：

```text
xcwindfall.top
```

多个项目就用子域名：

```text
newapi.xcwindfall.top       → New API
sub2api.xcwindfall.top      → Sub2API
cliproxy.xcwindfall.top     → CLIProxyAPI
abc.xcwindfall.top          → 其他项目
```

不要用这种方式：

```text
xcwindfall.top/newapi
xcwindfall.top/sub2api
```

原因：很多 API 服务、OAuth 回调、WebSocket、前后端路径都容易出问题。子域名最稳。

## DNS 配置

去你的域名解析面板添加 A 记录：

| 主机记录 | 类型 | 值 |
|---|---|---|
| `newapi` | A | 你的服务器公网 IP |
| `sub2api` | A | 你的服务器公网 IP |
| `cliproxy` | A | 你的服务器公网 IP |
| `*` 可选 | A | 你的服务器公网 IP |

如果你服务器 IP 是：

```text
1.2.3.4
```

那就填：

```text
newapi  A  1.2.3.4
sub2api A  1.2.3.4
cliproxy A 1.2.3.4
```

检查 DNS 是否生效：

```bash
ping newapi.xcwindfall.top
ping sub2api.xcwindfall.top
ping cliproxy.xcwindfall.top
```

或者：

```bash
nslookup newapi.xcwindfall.top
```

## Caddy 配置

编辑：

```bash
sudo nano /etc/caddy/Caddyfile
```

写入：

```caddyfile
newapi.xcwindfall.top {
    encode gzip
    reverse_proxy 127.0.0.1:3000
}

sub2api.xcwindfall.top {
    encode gzip
    reverse_proxy 127.0.0.1:8081
}

cliproxy.xcwindfall.top {
    encode gzip
    reverse_proxy 127.0.0.1:8317
}
```

检查配置：

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
```

重载 Caddy：

```bash
sudo systemctl reload caddy
```

看日志：

```bash
sudo journalctl -u caddy -f
```

## Caddy 和 Docker 的关系

Caddy 在宿主机上跑。

Docker 项目只需要把服务映射到宿主机本地端口：

```yaml
ports:
  - "127.0.0.1:3000:3000"
```

Caddy 再转发：

```caddyfile
newapi.xcwindfall.top {
    reverse_proxy 127.0.0.1:3000
}
```

这就是完整链路：

```text
https://newapi.xcwindfall.top
  → Caddy 443
  → 127.0.0.1:3000
  → new-api 容器
```

## 新增一个项目怎么做

假设新增项目叫 `demo`，端口用 `9001`。

1. DNS 添加：

```text
demo.xcwindfall.top A 你的服务器 IP
```

2. 项目 compose 端口写：

```yaml
ports:
  - "127.0.0.1:9001:8080"
```

3. Caddyfile 加：

```caddyfile
demo.xcwindfall.top {
    encode gzip
    reverse_proxy 127.0.0.1:9001
}
```

4. 重载：

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

## 常见问题

### 访问域名打不开

先检查 4 个地方：

```bash
ping newapi.xcwindfall.top
sudo ss -lntp | grep 3000
curl -I http://127.0.0.1:3000
sudo journalctl -u caddy -n 100
```

### Caddy 证书申请失败

一般是：

1. DNS 没生效。
2. 服务器 80/443 没开放。
3. 域名解析到错 IP。
4. Caddyfile 写错。

### 项目本地能访问，域名不能访问

说明 Docker 项目大概率没问题，重点查 Caddy：

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
sudo journalctl -u caddy -f
```
