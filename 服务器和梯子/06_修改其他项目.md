# 6. 修改其他项目

这个笔记用于以后新增项目、二创项目、fork 项目、自写项目。

## 1. 新增项目通用流程

假设项目名叫：

```text
demo
```

域名：

```text
demo.xcwindfall.top
```

宿主机端口：

```text
9001
```

目录：

```bash
sudo mkdir -p /opt/projects/demo
sudo chown -R $USER:$USER /opt/projects/demo
cd /opt/projects/demo
```

## 2. 通用 `docker-compose.yml` 模板

如果项目已经有镜像：

```yaml
services:
  demo:
    image: 你的镜像:latest
    container_name: demo
    restart: unless-stopped
    ports:
      - "127.0.0.1:9001:8080"
    environment:
      TZ: Asia/Shanghai
    volumes:
      - ./data:/app/data
```

意思：

```text
9001 = 服务器本地端口
8080 = 容器内部端口
```

Caddy 只认宿主机端口：

```caddyfile
demo.xcwindfall.top {
    encode gzip
    reverse_proxy 127.0.0.1:9001
}
```

## 3. fork / 二创项目模板

如果你拉源码自己改：

```bash
cd /opt/projects/demo
git clone 你的仓库地址 src
```

`docker-compose.yml`：

```yaml
services:
  demo:
    build:
      context: ./src
      dockerfile: Dockerfile
    image: local/demo:dev
    container_name: demo
    restart: unless-stopped
    ports:
      - "127.0.0.1:9001:8080"
    environment:
      TZ: Asia/Shanghai
    volumes:
      - ./data:/app/data
```

启动：

```bash
docker compose up -d --build
```

以后改完代码更新：

```bash
cd /opt/projects/demo/src
git pull

cd /opt/projects/demo
docker compose up -d --build
docker compose logs -f
```

## 4. 修改项目端口

假设原来：

```yaml
ports:
  - "8080:8080"
```

改成：

```yaml
ports:
  - "127.0.0.1:9001:8080"
```

左边是宿主机端口，右边是容器端口：

```text
127.0.0.1:9001:8080
        ↑    ↑
      外面   容器
```

然后 Caddy 写：

```caddyfile
demo.xcwindfall.top {
    reverse_proxy 127.0.0.1:9001
}
```

重启：

```bash
docker compose up -d
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

## 5. 多项目端口分配建议

| 项目类型 | 建议端口 |
|---|---:|
| New API | 3000 |
| Sub2API | 8081 |
| CLIProxyAPI | 8317 |
| 二创项目 1 | 9001 |
| 二创项目 2 | 9002 |
| 二创项目 3 | 9003 |
| 测试项目 | 9101+ |

查看哪些端口已用：

```bash
sudo ss -lntp
```

## 6. 修改 Caddy 后必须执行

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

如果失败，看原因：

```bash
sudo journalctl -u caddy -n 100
```

## 7. 项目更新前备份

每个项目都建议有 `backups` 目录：

```bash
mkdir -p backups/$(date +%F_%H%M)
```

通用备份：

```bash
BACKUP_DIR=backups/$(date +%F_%H%M)
mkdir -p $BACKUP_DIR

tar czf $BACKUP_DIR/project_files.tar.gz \
  docker-compose.yml .env data config.yaml logs 2>/dev/null
```

如果有 PostgreSQL：

```bash
docker exec 容器名 pg_dump -U 用户名 数据库名 > $BACKUP_DIR/db.sql
```

## 8. 项目更新

镜像项目：

```bash
docker compose pull
docker compose up -d
docker compose logs -f
```

源码项目：

```bash
cd src
git pull

cd ..
docker compose up -d --build
docker compose logs -f
```

## 9. 回滚

如果更新后挂了：

```bash
docker compose down
```

恢复备份文件：

```bash
tar xzf backups/你的备份目录/project_files.tar.gz
```

重新启动：

```bash
docker compose up -d
```

如果数据库也更新坏了，恢复数据库备份。

## 10. 判断问题在哪

按这个顺序查：

```text
1. DNS 是否解析到服务器
2. Caddy 是否正常
3. 宿主机端口是否监听
4. Docker 容器是否运行
5. 项目日志是否报错
6. 数据库 / Redis 是否正常
```

对应命令：

```bash
ping demo.xcwindfall.top
sudo systemctl status caddy
sudo ss -lntp | grep 9001
docker compose ps
docker compose logs -f
```

## 11. 不要这样搞

不要把所有项目都放一个 compose 里。

不要把数据库端口暴露公网：

```yaml
ports:
  - "5432:5432"
```

不要所有项目都用同一个端口：

```yaml
ports:
  - "8080:8080"
```

不要直接公网访问项目端口：

```text
http://服务器IP:9001
```

应该统一用：

```text
https://demo.xcwindfall.top
```
