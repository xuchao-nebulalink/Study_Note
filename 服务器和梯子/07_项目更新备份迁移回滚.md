# 项目更新、备份、迁移、回滚统一笔记

> 所有项目通用。重点：更新前一定备份，尤其是有 PostgreSQL/Redis/SQLite 的项目。

## 1. 更新前先看项目目录

```bash
cd /opt/apps
ls
```

进入对应项目：

```bash
cd /opt/apps/newapi
```

查看 compose：

```bash
ls -lah
cat docker-compose.yml
```

## 2. 最简单的整目录备份

适合所有项目。

```bash
cd /opt/apps
tar czf 项目名_full_$(date +%Y%m%d_%H%M%S).tar.gz 项目名
```

例如 New API：

```bash
cd /opt/apps
tar czf newapi_full_$(date +%Y%m%d_%H%M%S).tar.gz newapi
```

Sub2API：

```bash
cd /opt/apps
tar czf sub2api_full_$(date +%Y%m%d_%H%M%S).tar.gz sub2api
```

CLIProxyAPI：

```bash
cd /opt/apps
tar czf cliproxyapi_full_$(date +%Y%m%d_%H%M%S).tar.gz cliproxyapi
```

## 3. PostgreSQL 备份

### 3.1 New API

```bash
cd /opt/apps/newapi
mkdir -p backups
BACKUP_TIME=$(date +%Y%m%d_%H%M%S)
docker compose exec -T newapi-postgres pg_dump -U newapi newapi > backups/newapi_pg_${BACKUP_TIME}.sql
```

### 3.2 Sub2API

```bash
cd /opt/apps/sub2api
mkdir -p backups
BACKUP_TIME=$(date +%Y%m%d_%H%M%S)
docker compose -f docker-compose.local.yml exec -T postgres pg_dump -U sub2api sub2api > backups/sub2api_pg_${BACKUP_TIME}.sql
```

### 3.3 其他项目

模板：

```bash
cd /opt/apps/项目名
mkdir -p backups
BACKUP_TIME=$(date +%Y%m%d_%H%M%S)
docker compose exec -T postgres pg_dump -U 数据库用户 数据库名 > backups/项目名_pg_${BACKUP_TIME}.sql
```

## 4. SQLite 备份

如果项目是 SQLite，一般就是复制 `.db` 文件。

```bash
cd /opt/apps/项目名
mkdir -p backups
cp data/*.db backups/db_$(date +%Y%m%d_%H%M%S).db
```

New API SQLite 常见：

```bash
cp data/one-api.db backups/one-api_$(date +%Y%m%d_%H%M%S).db
```

## 5. Redis 备份

如果 Redis 只是缓存，通常不需要单独备份。

如果 Redis 里有重要队列/状态，你可以停项目后打包 `redis_data`：

```bash
cd /opt/apps/项目名
docker compose down
tar czf backups/redis_data_$(date +%Y%m%d_%H%M%S).tar.gz redis_data
docker compose up -d
```

## 6. Docker 镜像项目更新

通用：

```bash
cd /opt/apps/项目名
docker compose pull
docker compose up -d
```

指定服务：

```bash
docker compose pull 服务名
docker compose up -d 服务名
```

看日志：

```bash
docker compose logs -f
```

## 7. build 本地源码项目更新

```bash
cd /opt/apps/项目名
git pull
docker compose up -d --build
```

如果你改过代码，先看：

```bash
git status
```

有本地修改时，不要直接 pull。

## 8. 二创项目更新流程

### 8.1 先备份

```bash
cd /opt/apps
tar czf myapp_full_$(date +%Y%m%d_%H%M%S).tar.gz myapp
```

### 8.2 提交自己的修改

```bash
cd /opt/apps/myapp
git status
git add .
git commit -m "backup my custom changes before upstream update"
```

### 8.3 拉上游

```bash
git remote -v
```

如果没有 upstream：

```bash
git remote add upstream https://github.com/原作者/项目.git
```

拉取：

```bash
git fetch upstream
```

合并：

```bash
git merge upstream/main
```

或者：

```bash
git merge upstream/master
```

### 8.4 重建

```bash
docker compose up -d --build
```

### 8.5 看日志

```bash
docker compose logs -f
```

## 9. 最稳的蓝绿更新法

适合不想把线上搞崩的项目。

```text
旧项目：myapp，端口 18081
新项目：myapp_new，端口 19081
```

流程：

```bash
cd /opt/apps
tar czf myapp_full_$(date +%Y%m%d_%H%M%S).tar.gz myapp

git clone https://github.com/作者/项目.git myapp_new
cd myapp_new
cp ../myapp/.env ./.env
cp -r ../myapp/data ./data
```

新项目 compose 端口：

```yaml
ports:
  - "127.0.0.1:19081:3000"
```

启动：

```bash
docker compose up -d --build
curl -I http://127.0.0.1:19081
```

Caddy 切流量：

```caddyfile
myapp.xcwindfall.top {
    reverse_proxy 127.0.0.1:19081
}
```

重载：

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

旧项目先不要删，观察几天。

## 10. 回滚方法

### 10.1 如果只是镜像更新坏了

把镜像 tag 改回旧版本：

```yaml
image: xxx:旧版本
```

然后：

```bash
docker compose up -d
```

### 10.2 如果整目录都备份了

```bash
cd /opt/apps
docker compose -f 项目名/docker-compose.yml down
mv 项目名 项目名_bad_$(date +%Y%m%d_%H%M%S)
tar xzf 项目名_full_时间.tar.gz
cd 项目名
docker compose up -d
```

Sub2API 要用：

```bash
docker compose -f docker-compose.local.yml up -d
```

### 10.3 PostgreSQL 恢复

先确认你真的要恢复，会覆盖当前数据。

```bash
cd /opt/apps/项目名
docker compose down
docker compose up -d postgres
cat backups/xxx.sql | docker compose exec -T postgres psql -U 用户 -d 数据库
 docker compose up -d
```

New API 示例：

```bash
cd /opt/apps/newapi
docker compose up -d newapi-postgres
cat backups/newapi_pg_时间.sql | docker compose exec -T newapi-postgres psql -U newapi -d newapi
docker compose up -d
```

Sub2API 示例：

```bash
cd /opt/apps/sub2api
docker compose -f docker-compose.local.yml up -d postgres
cat backups/sub2api_pg_时间.sql | docker compose -f docker-compose.local.yml exec -T postgres psql -U sub2api -d sub2api
docker compose -f docker-compose.local.yml up -d
```

## 11. 定期备份建议

每天凌晨 3 点备份所有项目目录：

```bash
sudo nano /opt/apps/backup_all.sh
```

写入：

```bash
#!/usr/bin/env bash
set -e
BACKUP_DIR=/opt/backups/apps
mkdir -p $BACKUP_DIR
cd /opt/apps
for app in newapi sub2api cliproxyapi; do
  if [ -d "$app" ]; then
    tar czf "$BACKUP_DIR/${app}_$(date +%Y%m%d_%H%M%S).tar.gz" "$app"
  fi
done
find $BACKUP_DIR -type f -name "*.tar.gz" -mtime +14 -delete
```

授权：

```bash
sudo chmod +x /opt/apps/backup_all.sh
```

加定时任务：

```bash
crontab -e
```

写入：

```cron
0 3 * * * /opt/apps/backup_all.sh >> /opt/backups/apps/backup.log 2>&1
```
