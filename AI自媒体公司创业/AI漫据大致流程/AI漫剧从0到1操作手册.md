# AI漫剧从0到1操作手册

## 一、完整流程总览

1. **定目标与平台**：明确发哪里、做多长、给谁看、想赚什么。
2. **对标拆解**：先拆同类内容的钩子、节奏、封面、标题和悬念结构。
3. **选择叙事模式**：先决定你这部漫剧靠旁白、对白还是字幕推进。
4. **定题材与项目定位**：选一个能连载的题材，确定一句话卖点。
5. **建立项目Bible**：把世界观、规则、角色关系、人设禁改项先定死。
6. **写季纲与前10集概要**：先设计长线悬念，再写单集。
7. **写单集剧本**：把剧情写成短视频能拍的脚本。
8. **角色与画风定标**：先定主角、配角、场景、色调，再批量做。
9. **拆生产分镜表**：把剧本拆成真正可排产、可追踪的镜头任务。
10. **批量出关键帧与普通镜头**：先关键帧，后补镜头。
11. **视频化与动效**：把静帧做成有节奏的漫剧画面。
12. **配音、音效、配乐**：让角色“说话”，补足氛围。
13. **剪辑包装与发布**：成片、封面、标题、简介、评论区引导。
14. **数据复盘与迭代**：根据完播、评论、转粉继续改下一集。

## 二、推荐AI分工

- **ChatGPT / GPT-5.4**｜定位、拆流程、写脚本、分镜表｜把想法整理成结构化方案；把故事改成镜头脚本；整理提示词库｜定位：总控型
- **Claude Opus 4.6**｜Bible、季纲、长线设定｜长文本设定统一；做角色关系、世界规则、前后呼应｜定位：长文型
- **OpenAI 图像**｜角色定稿、关键帧、精修｜先把角色脸、服装、表情、主场景定准｜定位：定稿型
- **Runway Gen-4/4.5**｜图生视频、一致性、多镜头延展｜让角色在不同镜头里尽量保持一致；做关键动态镜头｜定位：视频型
- **Gemini / Veo 3.1**｜少量高质量关键镜头｜做少数高价值镜头，不建议整集全靠它｜定位：高光镜头型
- **ElevenLabs**｜角色配音、旁白｜设计固定音色，做主角、配角、旁白三套声音｜定位：声音质感型
- **CapCut**｜快速配音、字幕、成片｜快速试样片、做字幕、音效、基础剪辑｜定位：效率型
- **Premiere Pro**｜精剪、声音修复、最终输出｜更稳地控节奏、音频清理、专业版成片｜定位：精修型

## 三、每一步具体怎么做


### 第1步：定目标与平台
- **输入物**：你的个人目标、可投入时间与预算、你擅长或想尝试的题材方向、可用工具能力、目标平台方向、已有参考账号或灵感素材。
- **目标**：先明确你做这个项目是为了涨粉、变现、接单，还是为了做样片和IP。目标不一样，流程细节也会不一样。
- **你要做**：确定发布平台（抖音/快手/B站/视频号等）、每集时长、更新频率、目标用户、变现方式、参考账号。
- **产出物**：一页项目定位表：一句话定位、目标平台、目标受众、时长、更新频率、变现方式、参考账号。
- **推荐AI**：ChatGPT / GPT-5.4 最适合。用它来把你脑子里的模糊想法整理成清晰的项目定位表；Claude 也可以做长版整理。
- **AI怎么用**：让AI输出“项目定位表 + 同类账号参考 + 你和别人差在哪”。
- **注意**：不要一开始就想“拍什么故事”，先想“这个号准备发给谁看”。

### 第2步：对标拆解
- **输入物**：第1步的项目定位表、目标平台候选账号列表、同题材样本池、你准备对标的时长范围与目标受众范围。
- **目标**：先拆清这个赛道里什么内容有人看，再决定自己怎么做，不要凭空想流程。
- **你要做**：找同平台、同题材、同时长的 20-50 条内容，拆前3秒钩子、叙事方式、冲突点、结尾悬念、封面、标题、更新频率、评论区高频点。
- **产出物**：对标拆解表：账号名、题材、时长、前3秒钩子、叙事方式、冲突点、结尾钩子、封面样式、标题模板、值得借鉴点、不要碰的坑。
- **推荐AI**：ChatGPT / Claude 都适合。ChatGPT 更适合把拆解内容整理成结构化表；Claude 更适合总结长样本共性。
- **AI怎么用**：把你收集到的样本发给AI，让它输出“高频钩子模式 + 常见节奏模板 + 你最适合切入的差异化方向”。
- **注意**：对标拆解不是抄别人的故事，而是先看清这个赛道到底什么东西有人看。

### 第3步：选择叙事模式
- **输入物**：第1步项目定位表、第2步对标拆解表、你的配音能力、是否做口型的成本判断、你能承受的角色数量与信息密度上限。
- **目标**：先确定你这部漫剧靠什么推进，是旁白、对白，还是字幕。
- **你要做**：在“旁白驱动、对白驱动、字幕驱动、第一人称自述”里先选主模式，再决定是不是混合使用。
- **产出物**：叙事模式说明：主叙事方式、台词密度、旁白比例、是否做口型、每集信息量上限。
- **推荐AI**：ChatGPT / Claude 都适合。ChatGPT 更适合把同一个剧情分别改写成不同叙事模式试样。
- **AI怎么用**：让AI把同一段剧情分别改成“旁白版、对白版、字幕版”，直接对比哪种最适合你的节奏和成本。
- **注意**：新手第一部作品通常优先“旁白驱动 + 少量对白”或“对白驱动但角色少”，别一开始就做多人高密度口型戏。

### 第4步：定题材与项目定位
- **输入物**：第1步项目定位表、第2步赛道拆解结论、第3步叙事模式说明、你能长期承受的角色数、场景数和制作成本上限。
- **目标**：挑一个能连载、能持续做的题材，不是只做一条爆款。
- **你要做**：从规则怪谈、都市悬疑、复仇反转、校园怪谈、夜班惊悚等方向中，选一个最适合短视频连载的题材。然后写出一句话卖点。
- **产出物**：题材卡：题材、核心冲突、主角目标、长期悬念、第一季能讲什么。
- **推荐AI**：ChatGPT / Claude 都适合。Claude更适合把题材的长线设定写扎实。
- **AI怎么用**：让AI一次给你 10-20 个题材方向，再从中筛 1 个最适合你起步的。
- **注意**：起步期别选世界观太大、角色太多、场景太散的题材。

### 第5步：建立项目Bible
- **输入物**：第4步题材卡、第1步项目定位表、第2步对标拆解表、第3步叙事模式说明、你已有的视觉参考与世界观灵感素材。
- **目标**：先把不能乱改的东西定死，后面才不会越做越乱。
- **你要做**：整理世界观规则、时间线、角色关系、角色禁改项、核心设定、镜头风格、视觉风格。
- **产出物**：项目Bible文档：世界规则、主角设定、配角设定、角色关系、禁改项、视觉说明。
- **推荐AI**：Claude Opus 4.6 最适合做长文统一；ChatGPT 适合把内容整理成结构化模板。
- **AI怎么用**：先用 ChatGPT 生成模板，再用 Claude 补完整设定和前后统一。
- **注意**：没有Bible，后面第3集以后角色和规则就很容易崩。

### 第6步：写季纲与前10集概要
- **输入物**：第5步项目Bible、第4步题材卡、第1步目标平台与时长要求、第2步高频钩子模式与节奏模板。
- **目标**：先设计长线悬念，再写单集，避免开头猛、后面水。
- **你要做**：写第一季主线、前10集梗概、每集钩子、每集结尾悬念。
- **产出物**：第一季总纲 + 前10集概要表。
- **推荐AI**：Claude 更适合做长线剧情；ChatGPT 更适合把概要改成短视频节奏。
- **AI怎么用**：让AI按“每集 60-90 秒”的长度去拆剧情，而不是写成小说。
- **注意**：先看每集结尾有没有悬念，再看中间细节。

### 第7步：写单集剧本
- **输入物**：第6步第一季总纲与前10集概要表、第5步项目Bible、本集时长限制、本集核心钩子与结尾悬念要求。
- **目标**：把故事改成“能做视频”的脚本，而不是小说。
- **你要做**：按场景写：地点、角色、冲突、台词、情绪、关键画面、结尾钩子。
- **产出物**：单集剧本：建议拆成 4-8 个场景，每场都有冲突和推进。
- **推荐AI**：ChatGPT / Claude 都可以。ChatGPT 更适合做“短视频脚本改写”。
- **AI怎么用**：告诉AI：不要写小说体，要写成短视频镜头剧本，前3秒必须有钩子。
- **注意**：每集的重点不是“讲完整”，而是“让人继续看下一集”。

### 第8步：角色与画风定标
- **输入物**：第5步项目Bible、第4步题材卡、第7步单集剧本、核心角色与主场景清单、你选定的视觉参考与封面风格方向。
- **目标**：先把角色和风格做成“标准件”，后面才能批量生产。
- **你要做**：做主角标准立绘、多角度参考、主要表情、服装方案、主场景样图、封面样式。
- **产出物**：角色图包、场景图包、画风说明、封面样式参考、正式角色ID/场景ID/服装版本标准件、资产总表。
- **推荐AI**：OpenAI 图像适合做角色定稿和精修；Runway 的参考图工作流适合后续做一致性延展。
- **AI怎么用**：先让图像AI把主角脸、服装、年龄感、表情做稳定；不要一开始混用太多模型。
- **注意**：这一步没做好，后面每一集的角色都会长得不一样。

### 第9步：拆生产分镜表
- **输入物**：第7步单集剧本、第5步项目Bible、第8步角色与画风标准件、第1步平台规格与时长要求、已有镜头模板与提示词模板。
- **目标**：把剧本变成真正能排产、能追踪、能复用的镜头任务单。
- **你要做**：除了写景别、机位、动作、情绪、台词、时长，还要给每个镜头标明角色参考图、场景ID、服装版本、素材类型、生成方式、提示词版本、是否口型、动态方式、文件命名、存储路径和当前状态。
- **产出物**：生产分镜表（可直接用于出图、动效、配音、剪辑协同）。
- **推荐AI**：ChatGPT 很适合做“剧本 -> 分镜表”转换。
- **AI怎么用**：让AI输出表格字段：镜头号、场次、时长、景别、机位、画面描述、角色动作、对白/旁白、场景ID、角色参考图ID、服装版本、素材类型、生成方式、提示词、负面提示词、口型、动态方式、字幕重点、音效/BGM、文件名、存储路径、状态、备注。
- **注意**：没有参考图ID、提示词版本、素材状态和文件命名规则，第3集以后素材一定会乱；分镜表必须是生产表，不只是创意草稿。

### 第10步：批量出关键帧与普通镜头
- **输入物**：第8步角色与画风标准件、第9步生产分镜表、正式角色ID与场景ID、提示词模板库、负面提示词模板、输出规格要求。
- **目标**：先把最重要的镜头做出来，再补一般镜头。
- **你要做**：先出封面、高潮镜头、主冲突镜头、结尾悬念镜头，再补普通过渡镜头。
- **产出物**：关键帧素材包 + 普通镜头素材包。
- **推荐AI**：OpenAI 图像负责关键帧和精修；Runway 可基于参考图继续生成一致性镜头。
- **AI怎么用**：先出 8-12 个关键镜头，确认角色没崩、风格没跑，再批量补齐。
- **注意**：不要按时间顺序一张张乱做，先做“记忆点画面”。

### 第11步：视频化与动效
- **输入物**：第10步关键帧素材包与普通镜头素材包、第9步分镜表中的动态要求、口型需求、镜头节奏要求和重点情绪镜头清单。
- **目标**：把静态漫画做成有节奏、有情绪、有推进的视频。
- **你要做**：做推拉、摇镜、分层视差、光效、雨雾、眨眼、嘴型、局部动作和少量完整视频镜头。
- **产出物**：动态镜头素材。
- **推荐AI**：Runway Gen-4/4.5 适合图生视频和一致性动态镜头；Gemini / Veo 3.1 适合少量高质量关键视频镜头。
- **AI怎么用**：大部分镜头用静帧+运镜解决，只有少数关键镜头再上完整视频生成。
- **注意**：第一部作品别追求每个镜头都真动，成本高且容易不稳。

### 第12步：配音、音效、配乐
- **输入物**：第7步单集剧本、第9步生产分镜表、第11步动态镜头素材、角色音色设定草案、音乐风格参考、平台版权要求。
- **目标**：让角色有声音，让氛围立起来。
- **你要做**：给主角、配角、旁白分别定音色；补脚步、门响、风声、环境声；加背景音乐。
- **产出物**：对白音轨、旁白音轨、音效包、背景音乐方案。
- **推荐AI**：ElevenLabs 适合角色化配音；CapCut 适合快速试声音；Runway 也可配合口型流程。
- **AI怎么用**：主角和旁白最好固定音色，别每集都换；先做旁白和1-2个主角声音即可。
- **注意**：配音一乱，整部漫剧的“专业感”会掉很多。

### 第13步：剪辑包装与发布
- **输入物**：第10步镜头素材、第11步动态镜头素材、第12步音频资产、第2步封面与标题拆解结论、第1步平台发布规格、发布时间计划。
- **目标**：把素材拼成“能发”的成片，不只是素材集合。
- **你要做**：完成剪辑、字幕、转场、节奏点、音画对齐、封面、标题、简介和置顶评论。
- **产出物**：最终成片 + 发布包装素材。
- **推荐AI**：CapCut 适合快速成片；Premiere 适合精剪和声音精修；ChatGPT 适合写标题、封面文案、评论区引导。
- **AI怎么用**：至少准备 2-3 个标题版本、2 个封面版本，发布时做 A/B 测试。
- **注意**：封面、标题、前3秒钩子，决定点击率；中段节奏和结尾悬念，决定完播率。

### 第14步：数据复盘与迭代
- **输入物**：第13步最终成片与发布包装素材、平台后台数据、用户评论与私信反馈、第1步项目目标、第2步对标拆解结论。
- **目标**：把作品做成稳定产线，而不是一集一集碰运气。
- **你要做**：看 3 秒停留、5 秒停留、完播率、评论率、收藏率、关注转化、下一集承接率。
- **产出物**：复盘表：有效镜头、无效台词、用户评论高频点、下一集优化项。
- **推荐AI**：ChatGPT / Claude 都适合做复盘总结。
- **AI怎么用**：把后台数据和用户评论喂给AI，让它总结“为什么有人刷走、为什么有人追更”。
- **注意**：复盘的目的不是复述数据，而是改第2集、第3集。

## 四、项目管理与资产管理规范
- **核心原则**：Bible管设定，资产总表管ID和版本，分镜表管生产。三张表别各写各的。
- **命名规则**：统一用 `项目_集数_场次_镜头_资产类型_ID_v版本号`，别再出现“最终版”“终版2”这种名字。
- **ID规则**：角色用 `CHR-xx`，场景用 `SCN-xx`，服装用 `CST-xx`，镜头用 `SH-xx`，提示词模板用 `PRM-xx`，音色用 `VOC-xx`。
- **状态规则**：素材至少分成 `draft / review / locked / deprecated` 四种状态。进入成片的素材一律标成 `locked`。
- **资产总表**：至少记 8 个字段：资产ID、资产名称、资产类型、版本号、状态、存储路径、首次启用集数、关联镜头。
- **变更规则**：只要改角色脸、服装、世界规则、主场景、主音色，先改 Bible 和资产总表，再回写分镜和素材。
- **文件夹结构**：
```text
AI漫剧项目/
  00_项目管理/
  01_Bible/
  02_季纲与概要/
  03_剧本/
  04_角色资产/
  05_场景资产/
  06_分镜表/
  07_镜头素材/
  08_动态视频/
  09_音频/
  10_剪辑工程/
  11_发布素材/
  12_数据复盘/
```
- **最低要求**：进入批量生产前，每个镜头至少要有镜头号、角色ID、场景ID、版本号、状态、提示词和存储路径。

## 五、第一部作品怎么起步最稳
- 题材优先：规则怪谈、夜班惊悚、都市悬疑、校园怪谈。
- 对标优先：先拆 20-50 条同赛道内容，再动手写自己的。
- 时长优先：每集 60-90 秒，竖屏优先。
- 叙事方式：新手优先“旁白驱动 + 少量对白”或“角色少的对白驱动”。
- 角色数量：2-4 个核心角色就够了。
- 场景数量：2-3 个主场景，方便复用。
- 启动方式：先做 3 集样片，不要直接铺 20 集。
- 镜头策略：70% 静帧+运镜，20% 局部动态，10% 高价值视频镜头。

## 六、最小可执行交付清单
1. **项目定位表**：确定平台、时长、用户、题材、变现方式
2. **对标拆解表**：前3秒钩子、叙事方式、封面、标题、结尾悬念
3. **叙事模式说明**：旁白/对白/字幕比例，是否做口型
4. **项目Bible**：世界规则、角色关系、禁改项
5. **前10集概要**：每集一句钩子和一句悬念
6. **第1集正式剧本**：按短视频格式，不写小说体
7. **资产总表**：角色ID、场景ID、服装版本、音色ID、资产状态、路径
8. **角色图包**：主角立绘、多角度、表情、服装
9. **主场景图包**：主空间、氛围、色调、道具
10. **第1集生产分镜表**：镜头号、时长、动作、台词、参考图ID、提示词版本、存储路径、素材状态
11. **关键镜头素材**：先做 8-12 个高价值镜头
12. **音轨方案**：旁白、主角音色、基础音效
13. **样片成片**：先完成第1-3集中的至少1集
